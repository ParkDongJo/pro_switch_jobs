
# 테스트를 작성해야하는 이유
- 사업의 신뢰성
- 높은 유지보수성
	- 리팩터링을 할 때, 테스트 코드가 있으면 구현된 코드를 바꾸는데 있어서 안정적으로 코드를 변경할 수 있다.
	- 이때 테스트 코드는 너무 구현코드에 의존성이 강하게 엮여 있으면, 결국 테스트 코드도 바뀌어야 한다. 즉, 느슨한 테스트코드가 필요하다.
- 코드 품질 향상
	- 테스트 친화적인 코드는 코드의 역할이 잘 나눠져있을 확률이 높다
	- 테스트 코드는 웹접근성을 높이는데도 기여한다. 웹접근성이 지켜지지 않은 코드는 테스트 코드 대상을 찾는데 어려움을 겪는다.
- 협업 향상
	- 테스트 코드는 우수한 사양서가 될 수 있다. 기능이 어떤 기능을 제공하고, 어떤방식으로 작동하는지 파악할 수 있다.
	- 사양서대로 구현되어 있는지 테스트 코드로 먼저 확인 할 수 있다.
- 회귀테스트를 줄여준다.
	- 모듈이 많아지면, 모듈 간 의존관계의 영향을 검증하고자 자주 회귀테스트 하게 된다. 테스트 코드가 있으면 이 비용을 줄일 수 있다.

> 회귀 테스트란?
> 회귀 테스트(Regression Testing)는 새로운 기능 개발이나 버그 수정 후, 이전에 정상적으로 동작하던 기존 기능에 문제가 발생하지 않았는지 확인하는 소프트웨어 테스트 방식

- **기능 회귀 테스트**: 버튼 클릭, 폼 입력 등 UI에서 기대한 동작이 동일한지 확인.
- **시각적 회귀 테스트**: 코드 변경 전후의 UI(화면)를 이미지로 캡처해 변화가 없는지 비교
- **E2E(End-to-End) 테스트**
