## 시각적 회귀 테스트와 스냅숏 테스트
스냅숏 테스트 - 클래스 속성의 변경으로 스타일이 변경되는 것을 알수 있다.
시각적 회귀 테스트 - 전역에 걸친 모든 시각적 테스트를 대체할 수 있다.

## 어떻게?
실제 브라우저에 렌더링된 화면을 비교하는 것이 가장 신뢰도가 높다. 특정 시점에 캡쳐한 이미지들을 픽셀 단위로 비교할 수도 있다.

세세하게 검출하고 싶다면 시각적 회긔 테스트를 UI 컴포넌트 단위로 실시 해야한다.

작은 크기, 중간 크기의 UI 컴포넌트를 스토리로 등록해두면, 시각적 회귀 테스트에도 활용할 수 있다.


## reg-cli 로 이미지 비교 (로컬)
스샙샷 이미지를 비교하는 도구

- actual
- expected
- diff

## Story-cap 으로 캡쳐
스토리북에 등록된 스토리를 모두 캡처하는 도구



## reg-suit 도입
reg-cli 로 로컬에서 시각적 회귀 테스트를 실시할 수 있다. 하지만 원격으로 테스트를 자동화 하려면 github 과 연동하여 reg-suit 으로 자동화를 해야한다.

- 설정파일 생성
- reg-suit Client Id 취득
- 오차범위 설정
- AWS S3 버킷 만들어놓기
- Github Action 에 reg-suit 연동
- Github Action 코드 작성


## 시각적 회귀 테스트 를 통한 활용
적극적인 리펙터링
안정적인 테스트 자동화
